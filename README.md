# Nervus · 从未停止存在的系统

> 一个持续存在的个人智能体。它不路由数据，它理解你。

---

## 这是什么

Nervus 不是一个 App，不是一个平台，不是一个助手。

**它是一个持续运行在你设备上的个人智能体——它唯一的工作，是持续理解你是谁。**

它 24 小时不间断地观察你的生活，建立对你这个人的推断模型。你装的每一个 App，都只是它的一个投影界面——展示它对你某个维度的理解。

```
你拍了一张炸鸡照片
  → 不是"Calorie App 收到了一张照片"
  → 而是"Nervus 更新了对你今天营养状态的推断，Calorie App 展示这个推断"

你开了 4 个连续的会
  → 不是"Meeting App 记录了4次会议"
  → 而是"Nervus 感知到你今天认知负荷极高，主动预判你晚饭可能会吃垃圾食品"

你连续两周睡眠变差
  → 不需要任何 App 告诉你
  → Nervus 自己发现了这个规律，在它变成问题之前提醒你"
```

**它从未停止存在过。你不需要管理它，它在管理对你的理解。**

## Personal Model：系统的核心

Personal Model 是 Nervus v2 的真正主角。它维护着若干**维度**——对你这个人某一方面的持续推断：

| 类别 | 维度 | 含义 |
|------|------|------|
| 身体健康 | `nutrition_24h` | 过去24小时营养摄入与质量推断 |
| 身体健康 | `sleep_quality` | 睡眠规律与质量趋势 |
| 身体健康 | `activity_today` | 今日活动水平 |
| 认知状态 | `cognitive_load_now` | 当前认知负荷（会议密度、任务压力） |
| 认知状态 | `stress_indicator` | 综合压力信号 |
| 知识兴趣 | `active_topics` | 当前正在学习或关注的话题 |
| 时间习惯 | `daily_routine` | 一天的时间分配模式 |
| 时间习惯 | `upcoming_context` | 未来24小时的预期状态 |
| 社交 | `social_rhythm` | 社交互动密度和模式 |

每个维度包含：**当前推断值 + 置信度 + 来源事件追溯 + 历史快照**。

维度不是由 App 填写的数据库——它是本地模型持续推断的结果，随感知事件实时更新。

**跨维度推断（Insight Engine）**是 v2 独有的能力：

```
"你在高会议密度的工作日，营养质量平均比其他天低 0.3 分"
"根据本周睡眠趋势和明天的日历密度，预测明天压力较高，建议提前处理待办"
"今日行为明显偏离近两周规律，可能处于异常状态"
```

这些洞察不需要用户主动查询——Insight Engine 每小时运行，自动发现和推送。

## v2 vs v1：为什么要重新设计

v1 已在设备上稳定运行。v2 是一次**架构范式的根本转变**，不是功能迭代。

### 核心哲学差异

| | v1 | v2 |
|--|----|----|
| **系统的主角** | Arbor Core（路由器） | Personal Model（推断引擎） |
| **App 的角色** | 接收事件、独立处理、自行推断 | Personal Model 的 UI 投影 |
| **智能在哪里** | 分散在每个 App 内部 | 集中在 Personal Model，跨域理解 |
| **Memory 的定位** | App 写入的事件日志 | Personal Model 的历史快照 |

### 数据流对比

**v1**：
```
感知事件
  → Arbor Core 路由
    → Calorie App 收到"食物照片"事件 → Calorie App 自己做营养分析 → 写入自己的数据库
    → Meeting App 收到"录音"事件 → Meeting App 自己转写整理 → 写入自己的数据库
    → 两个 App 的数据互不相知
```

**v2**：
```
感知事件
  → Model Updater 更新 Personal Model 维度
    → nutrition_24h 更新 → Calorie App 收到通知，展示新的营养状态
    → cognitive_load 更新 → Meeting App 收到通知，展示今日压力预警
    → Insight Engine 发现两个维度的关联规律
```

### 关键能力对比

| 能力 | v1 | v2 |
|------|----|----|
| 单 App 内查询历史 | 支持 | 支持 |
| **跨 App 规律发现** | 不支持（数据孤岛） | **支持**（统一维度） |
| **主动预判和建议** | 不支持 | **支持**（Insight Engine） |
| 卸载 App 后数据 | 随 App 消失 | **维度历史永久保留** |
| 新 App 接入时 | 从零积累数据 | **立刻继承完整历史** |
| 自然语言问"我最近状态怎样" | 不支持 | **支持**（Personal Model API） |

### v1 能做到、v2 同样能做到的事

v2 保留了 v1 的快循环（Arbor Core），v1 的所有实时响应能力在 v2 中完整保留，只是新增了慢循环和 Personal Model 层。

| | 本地小模型（常驻） | 云端大模型（按需） |
|--|--|--|

## 设计灵感

Nervus v2 的核心范式转变——**从"工具路由中心"到"持续推断的个人智能体"**——受到了一套 AI 原生组织设计框架的深刻启发。

这套框架由一位律师朋友主导设计，用于重构法律服务组织形态。其核心论断是：

> **AI 不应该是外挂的工具，而应该是组织的基因与基础设施。**
> 不是"用 AI 的组织"，而是"围绕 AI 重新设计的组织"。

这个思路直接触发了 Nervus v2 的架构翻转：

| OneLaw 的组织设计观 | 迁移到 Nervus v2 |
|---------------------|-----------------|
| AI 是组织的基因，不是工具 | Personal Model 是系统的核心，不是辅助模块 |
| 从"个人经验"到"系统智能" | 从"App 各自推断"到"统一维度，跨域理解" |
| 节点归档后经验数据永久保留 | App 卸载后维度历史完整保留，理解不随 App 消失 |
| 新成员加入立即继承组织知识 | 新 App 接入立即继承完整的 Personal Model 历史 |
| 激励可见化：把隐性协作价值量化 | 维度可见化：把隐性的个人规律推断成可查询的状态 |

该框架由「未来法域」团队（OneLaw Hub）公开发布，见白皮书系列：
《ALL未来所 | OneLaw——革命级AI原生"超级物种"诞生，自演进可迁移AI组织生态框架建构白皮书》
关注微信公众号「未来法域」可获取原文。

Nervus 将同一套范式从"专业服务组织"迁移到了"个人神经系统"。感谢这套框架提供的思想原点。
